# dose-optim-frontend

Clinical web application for AI-driven stroke rehabilitation dose optimization. Clinicians enter per-patient data and receive personalized weekly treatment dose recommendations with confidence intervals, generated by a Gaussian CEM optimizer and Monte Carlo uncertainty estimation running in the backend.

> **This is the frontend repository.** The backend (Python model + FastAPI) is maintained in a private repository. The two services communicate over HTTP — no Python code lives here.

## Status

The current code is adapted from [`precision-rehab-demo`](https://github.com/Lydia-Ye/precision-rehab-demo), a standalone frontend-only demo that ran entirely on mock data. The UI components are retained as-is; the work remaining is replacing the mock logic with real API calls to the FastAPI backend (see [TODOs](#todos) below).

## What it does

- **Patient dashboard** — list and manage enrolled rehabilitation patients
- **Dose recommendation view** — displays optimized weekly treatment schedules with uncertainty estimates per patient
- **Outcome trajectory charts** — visualizes predicted MAL, UEFM, and WMFT recovery trajectories over the treatment horizon
- **Manual schedule comparison** — clinicians input an alternative schedule and compare predicted outcomes side-by-side
- **Observation recording** — logs observed outcomes that feed back into the backend's parameter estimation pipeline

| CI/CD | GitHub Actions → GCP Artifact Registry → Cloud Run |


## Architecture

```
Browser
    │  HTTPS
    ▼
Next.js frontend          ← this repo
    │  HTTP (proxy routes in src/app/api/)
    ▼
FastAPI backend            ← private repo
    │
    ▼
PostgreSQL
```

In production both containers are deployed independently on GCP Cloud Run.

## Project structure

```
src/
  app/
    api/          ← Next.js route handlers (currently mock, to be replaced with proxies)
    (pages)/      ← App Router page components
  components/     ← Shared UI components
  types/          ← TypeScript types (to be updated to match backend API schemas)
  mock/           ← Mock data from demo app (used until backend integration is complete)
```
