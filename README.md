# dose-optim-frontend

Clinical web application for AI-driven stroke rehabilitation dose optimization. Clinicians enter per-patient data, run the predictive model, and receive personalized weekly treatment dose recommendations with confidence intervals generated by a CEM optimizer and Monte Carlo uncertainty estimation in the backend.

> **This is the frontend repository.** The backend (Python model + FastAPI) is maintained in a private repository. The two services communicate over HTTP — no Python code lives here.

## Status

The current code is adapted from [`precision-rehab-demo`](https://github.com/Lydia-Ye/precision-rehab-demo), a standalone frontend-only demo that ran entirely on mock data.

## What it does

- **Patient dashboard** — list and manage enrolled rehabilitation patients
- **Trajectory chart** — visualizes the model's predicted MAL recovery curve from a subject's actual dose history
- **Optimal schedule** — runs CEM optimization for a subject and overlays the optimal trajectory on the chart
- **Manual schedule comparison** — enter a custom dose schedule and compare predicted outcomes against the optimal
- **Active patient workflow** — add new patients, run predictions, and record observed outcomes

## Architecture

```
Browser
    │  HTTPS
    ▼
Next.js frontend          ← this repo
    │  HTTP (proxy routes in src/app/api/)
    ▼
FastAPI backend            ← private repo
    │
    ▼
PostgreSQL
```

In production both containers are deployed independently on GCP Cloud Run.

## Project structure

```
src/
  app/
    api/
      patients/         ← GET list, POST create, PUT update
      patients/[id]/    ← GET single patient with trajectory
      results/          ← POST optimize → recommended schedule
      manual-predict/   ← POST manual schedule prediction
      patientinfo/      ← PUT update patient horizon
    patient/
      page.tsx          ← Patient dashboard (list view)
      [id]/page.tsx     ← Patient detail page
  components/
    PastPatientPage.tsx    ← Trial subject view with optimize/manual-predict
    NewPatientPage.tsx     ← Active patient view
    CurrentPredictChart.tsx
    PredictChart.tsx
  lib/
    backend.ts          ← Shared adapter: BackendPatient types, toFrontendPatient, enrichWithTrajectory, fetch helpers
  types/                ← TypeScript interfaces (Patient, request/response shapes)
```

## Running locally

**Prerequisites:** the backend stack must be running first.

```bash
npm install
npm run dev
```

The app runs on `http://localhost:3000`. `BACKEND_URL` in `.env.local` points to `http://localhost:8000`.

## Environment variables

| Variable | Default | Description |
|---|---|---|
| `BACKEND_URL` | `http://localhost:8000` | FastAPI backend base URL (server-side only) |

## Loading demo data

After the backend is up, import the 395 trial subjects from the CSV:

```bash
curl -X POST http://localhost:8000/v1/admin/import-csv \
  -F "file=@path/to/all_subjects_parameters.csv"
```

Subjects appear in the patient dashboard as past patients. Click any subject to view their predicted recovery trajectory and run the model.
